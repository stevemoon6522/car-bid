## Algorithm Detail (User Decisions 반영)

### 입력 (사용자 / 직원)
```
{
  "차명":     "K5",
  "모델명":   "K5 3세대(19년~현재",
  "년식":     2021,
  "주행거리": 80000,
  "등급_앞":  "B",        # 사고 컨디션 등급만 (앞글자)
  "색상":     "검정",
  "옵션":     ["네비", "스마트키", "선루프"]  # 보유 옵션 리스트
}
```

### 매칭 절차

```
1. WHERE 차명 = input.차명 AND 모델명 = input.모델명
   AND 낙찰/유찰 = '낙찰'
   AND 경매일자 >= today - 3months

2. IF count(후보) == 0:
     경매일자 >= today - 6months
     IF count == 0: return "비교 가능한 최근 낙찰 이력 없음"

3. 각 후보 행에 대해 유사도 점수 계산:
   - 등급_앞 점수 = match_grade(input.등급_앞, row.등급[0]) × 40
     - 정확 일치: 40
     - 한 단계 차이 (A↔B, B↔C, C↔D): 20
     - 두 단계 이상: 0
   - 년식 점수 = year_score(input.년식, row.년식) × 40
     - 정확 일치: 40
     - 1년 차이: 32, 2년 차이: 24, 3년: 16, 4년: 8, 5년+: 0
   - 주행거리 점수 = mileage_score(input.주행거리, row.주행거리) × 20
     - 5% 이내: 20
     - 10% 이내: 16, 20%: 12, 30%: 8, 40%: 4, 50%+: 0
   - 총점 = 등급 + 년식 + 주행거리 (0~100)

4. 상위 5건 선택 (총점 순)

5. 각 선택 건의 낙찰가에 가격 보정 적용:
   adjusted_price = row.낙찰가
   IF "선루프" in row.옵션 AND "선루프" NOT in input.옵션:
       adjusted_price -= 100  (만원)
   IF row.색상 == "흰색" AND input.색상 != "흰색":
       adjusted_price -= 100  (만원)
   # 향후 추가 보정 규칙 확장 가능

6. recommended_bid = adjusted_price × 0.85

7. 결과 반환:
   - 5개 후보 카드 (유사도 + breakdown + 원가 + 보정 + 추천가)
   - 추천 입찰가 통계 (min / median / mean / max)
```

### 알고리즘 최종 사양

**필터 (필수):**
- 차명 = input.차명 (exact)
- 모델명 = input.모델명 (exact)
- 낙찰/유찰 = "낙찰"
- 경매일자: 최근 3개월 → 0건이면 3~6개월 → 6개월도 0건이면 "비교 불가" 반환

**유사도 점수 (랭킹용, 0~100):**
- 등급 (첫 글자 A/B/C/D 만): **40점**
  - 동일: 40, 1단계 차이: 28, 2단계: 12, 3단계: 2
- 년식: **40점**
  - 동일: 40, 1년: 32, 2년: 24, 3년: 16, 4년: 8, 5년+: 0
- 주행거리: **20점**
  - 5% 이내: 20, 10%: 16, 20%: 12, 30%: 8, 40%: 4, 50%+: 0
- 색상, 옵션, 연료, 변속기: **점수 무관** (가격 보정에서만 처리)

**가격 보정 (각 후보 낙찰가에 적용):**
- 엑셀 옵션 컬럼에 "선루프" 부분 문자열 포함 AND 입력 옵션에 "선루프" 부분 문자열 없음 → 낙찰가 -100만원
- 엑셀 색상 == "흰색" AND 입력 색상 != "흰색" → 낙찰가 -100만원
- 두 조건 동시 만족 시 → -200만원 (누적 적용)

**추천 입찰가:**
- recommended_bid = 보정된 낙찰가 × 0.85 (-15%)
- 라벨: "보수 추천가 (Conservative)"

**출력:**
- 상위 5건 (유사도 점수 순)
- 각 건: 매칭 점수 + breakdown + 원 낙찰가 + 보정 내역 + 보정 후 가격 + 추천 입찰가
- 요약: 5건 평균/중앙값/min/max 추천 입찰가
- 신뢰도 표시: 데이터 기간 (3개월 ✅ / 6개월 ⚠)

## Database Schema (Supabase)

테이블 prefix: `car_*` (다른 앱과 충돌 방지).

```sql
create table public.car_auctions (
  id bigint generated by default as identity primary key,
  -- Source columns (verbatim from Excel '이번주거래(누적)')
  auction_date date not null,                -- 경매일자
  vehicle_class text,                        -- 차종 (RV/세단/SUV)
  maker text,                                -- 제작사
  car_name text not null,                    -- 차명 (e.g. K5)
  model_name text not null,                  -- 모델명 (e.g. K5 3세대(19년~현재)
  full_title text,                           -- 차종(*) full descriptive
  year int,                                  -- 년식
  first_registered date,                     -- 최초등록일
  transmission text,                         -- 변속기
  engine_code text,                          -- 엔진형식코드
  engine_cc int,                             -- 배기량
  fuel text,                                 -- 연료
  grade text,                                -- 등급 (2글자, e.g. "BC")
  color text,                                -- 색상
  mileage_km int,                            -- 주행거리
  search_category text,                      -- 검색구
  options text,                              -- 옵션 (comma-separated)
  product_type text,                         -- 상품구분
  accident_count int,                        -- 사고품횟수
  inspection_notes text,                     -- 검사내용
  start_price int,                           -- 시작가 (만원)
  bid_price int,                             -- 응찰가 (만원)
  final_price int,                           -- 낙찰금액VAT포함 (만원) ⭐
  status text not null,                      -- 낙찰/유찰
  -- Computed
  grade_first char(1),                       -- 등급 첫 글자 (A/B/C/D)
  is_outlier boolean default false,          -- mileage < 100 OR > 500,000
  row_hash text unique,                      -- composite dedup hash
  ingested_at timestamptz default now()
);

create index idx_car_auctions_lookup on public.car_auctions
  (car_name, model_name, status, auction_date desc);
create index idx_car_auctions_year on public.car_auctions (year);
create index idx_car_auctions_grade_first on public.car_auctions (grade_first);
```

**Dedup hash 정의:**
```python
hash = sha256(f"{auction_date}|{car_name}|{model_name}|{year}|{mileage_km}|{grade}|{color}|{final_price}")
```

**Ingest 정책:**
- 매주 새 엑셀 받으면 `scripts/ingest_excel.py path/to/new.xlsx --upsert`
- ON CONFLICT (row_hash) DO NOTHING (기존 데이터 보존)
- ingest 후 로깅: parsed N, upserted M, duplicates (N-M)
